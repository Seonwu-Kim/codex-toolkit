# Project Report Outline

## 1. 요약

State objective, reporting period, overall status, key outcome, and the most
important risk or decision.

## 2. 요약 지표

Use a table with metric, baseline, target, actual, status, and evidence source.

## 3. 마일스톤

Use a table with milestone, planned date, actual/forecast date, owner, status,
variance, and note.

## 4. 구현 결과

Describe delivered capabilities and observable outcomes. Link each result to
commits, artifacts, demos, or acceptance evidence where available.

## 5. 테스트 결과

Record scope, environment, command or method, pass/fail counts, defects,
coverage limitations, and residual risk.

## 6. 이슈 및 리스크

State impact, likelihood, mitigation, owner, and decision deadline.

## 7. 결정 사항

Capture decisions that changed scope, schedule, architecture, or release
criteria.

## 8. 다음 작업

Prioritize concrete tasks with owner, target date, dependency, and completion
criterion.
